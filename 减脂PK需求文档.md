# 减脂PK · 需求文档 (PRD)

> 版本: v1.0 | 日期: 2026-07-14 | 设计风格: A+C融合版

---

## 1. 项目概述

### 1.1 产品名称
**减脂PK** — 减脂期记录与好友对战平台

### 1.2 产品定位
一款面向减脂/健身人群的 Web 应用，帮助用户记录每日饮食卡路里、体重变化，并与好友进行限时减脂 PK 对战，通过社交激励提升坚持动力。

### 1.3 核心价值
- **精准记录**：食物卡路里计算 + 体重随时记录 + 每日身体情况日志
- **社交激励**：两人 PK 对战模式，设定时间段和目标，互相监督
- **数据可视化**：体重趋势、营养摄入、PK 进度一目了然
- **简洁界面**：A+C 融合风格，大留白 + 紫色主调，干净不花哨

### 1.4 部署方式
Docker 容器化部署，支持一键 docker-compose 启动（前端 + 后端 + 数据库）。

## 2. 用户角色

| 角色 | 说明 | 核心场景 |
|------|------|----------|
| 普通用户 | 注册登录后使用全部功能 | 记录饮食、记录体重、发起/接受PK、查看趋势 |
| 访客 | 未登录访问 | 仅可查看落地页/介绍页 |

> 本系统为轻量级应用，不设管理员角色。用户自行管理自己的数据。

## 3. 功能需求

### 3.1 用户系统

#### 3.1.1 注册与登录
- 用户名 + 密码注册
- 登录后获取 JWT Token，后续请求携带 Token
- 支持修改密码、修改昵称、上传头像

#### 3.1.2 个人资料
- 基础信息：昵称、性别、身高、年龄
- 减脂目标：目标体重、每日卡路里目标
- BMI 自动计算（根据身高体重）

#### 3.1.3 好友系统
- 通过用户名搜索添加好友
- 好友列表查看
- 发起 PK 对战（需好友接受）

---

### 3.2 饮食记录与卡路里计算

#### 3.2.1 食物数据库
- 内置常见食物数据库（名称、分类、每100g 热量/蛋白质/碳水/脂肪）
- 食物分类：早餐常用、午餐常用、加餐、水果、饮品、主食、肉蛋、蔬菜
- 支持搜索食物名称模糊匹配
- 支持用户自定义食物条目（保存到个人食物库）

#### 3.2.2 记录饮食
- 选择食物 → 输入份量（克数或份） → 自动计算热量和三大营养素
- 按餐次分类：早餐 / 午餐 / 晚餐 / 加餐
- 记录时间默认为当前，可手动调整
- 当日总摄入 = 各餐次之和

#### 3.2.3 卡路里总览
- 当日已摄入热量、剩余可吃热量、运动消耗热量
- 三大营养素进度条（蛋白质 / 碳水 / 脂肪，显示已摄入/目标值）
- 历史日期饮食记录查看

#### 3.2.4 快捷记录
- 常吃食物快捷面板（最近记录过的食物一键添加）
- 复制昨日饮食到今日

---

### 3.3 体重记录

#### 3.3.1 随时记录
- 输入体重数值（支持一位小数，单位 kg）
- 每日可记录多次，以最新一次为当日体重
- 可选记录体脂率、肌肉量（选填）

#### 3.3.2 体重趋势
- 折线图展示体重变化（7天 / 30天 / 全部）
- 显示较昨日变化、本周变化、本月变化
- 目标体重线标记
- 迷你柱状图预览（首页体重卡片内嵌）

#### 3.3.3 身体数据日志
- 每日可填写文字日志（身体感受、运动情况、备注）
- 日历视图：有记录的日期标记圆点
- 点击日期查看当日详情

---

### 3.4 PK 对战模式

#### 3.4.1 发起对战
- 用户向好友发起 PK 邀请
- 设定参数：
  - 对战名称（如"7月减脂挑战赛"）
  - 起止日期（时间段）
  - 目标：减重目标 kg 数（双方各自的目标）
  - 赌注/奖励（文字描述，如"请客一餐"）
- 好友接受后对战生效

#### 3.4.2 对战面板
- 双方头像、昵称、当前已减重 kg 数
- 达成百分比（已减 / 目标）
- VS 对比分隔，领先方高亮
- 进度条对比（双方占比条）
- 剩余天数倒计时
- 每日打卡提醒：双方可看到对方今日是否已记录体重

#### 3.4.3 对战状态
- 状态流转：待接受 → 进行中 → 已结束
- 对战结束后自动判定胜负（达成目标比例高者胜）
- 可提前结束（双方同意）

#### 3.4.4 历史对战
- 查看所有历史对战记录
- 胜负统计（总场数、胜场、负场）

---

### 3.5 首页仪表盘

#### 3.5.1 布局（对应A+C融合设计）
- **顶部**：头像 + 问候语 + 日期 + 通知/设置入口
- **卡路里环**：大圆环展示当日已摄入/目标，中心显示数字
- **三栏统计**：剩余可吃 / 已摄入 / 运动消耗
- **营养进度条**：蛋白质 / 碳水 / 脂肪三行进度
- **体重卡片**：渐变色背景，当前体重 + 变化 + 迷你趋势图
- **今日饮食**：按餐次分组的食物列表
- **PK 面板**：双方对比 + 进度条 + 剩余天数
- **底部导航**：首页 / 饮食 / 体重 / 对战 / 我的
- **悬浮按钮(+)**：快捷添加饮食/体重

## 4. 非功能需求

### 4.1 性能
- 页面首屏加载 < 2s
- API 响应时间 < 300ms（不含网络）
- 支持 100 并发用户

### 4.2 兼容性
- 移动端优先（375px 基准），适配平板和桌面端
- 支持 Chrome / Safari / Firefox / Edge 最新版
- PWA 支持（可选，可添加到手机主屏幕）

### 4.3 安全
- 密码 bcrypt 加密存储
- JWT Token 认证，有效期 7 天
- 用户只能查看/操作自己的数据
- PK 数据仅对战双方可见

### 4.4 可用性
- 界面简洁干净，关键操作 ≤ 3 步完成
- 无冗余弹窗，操作反馈通过页面内提示
- 离线时已加载数据可查看（只读模式）

## 7. 技术栈

### 7.1 前端
- **框架**: Vue 3 + Composition API
- **构建**: Vite
- **UI**: 自定义组件（A+C融合风格，紫色主调 #6c5ce7）
- **图表**: Chart.js（体重折线图）
- **HTTP**: Axios
- **状态管理**: Pinia
- **路由**: Vue Router

### 7.2 后端
- **框架**: Python FastAPI
- **ORM**: SQLAlchemy 2.0
- **数据库**: MySQL 8.0
- **缓存**: Redis（Token缓存 + 热数据）
- **认证**: JWT (python-jose)
- **密码**: bcrypt (passlib)

### 7.3 部署
- **容器**: Docker + Docker Compose
- **Web服务器**: Uvicorn (FastAPI ASGI)
- **静态文件**: Nginx 反向代理（前端静态 + API转发）

## 8. Docker 部署方案

### 8.1 容器编排（docker-compose）

三个容器：

| 容器 | 镜像 | 端口 | 说明 |
|------|------|------|------|
| frontend | nginx:alpine + 构建产物 | 80 → 8080 | 前端静态文件 + 反向代理 |
| backend | python:3.12-slim + FastAPI | 8000 | API服务 |
| database | mysql:8.0 | 3306 | 数据持久化 |

### 8.2 数据持久化
- MySQL 数据卷挂载: `/var/lib/mysql`
- 上传文件（头像等）: `/app/uploads`

### 8.3 环境变量
```
DB_HOST=database
DB_PORT=3306
DB_USER=fatloss
DB_PASSWORD=<your_password>
DB_NAME=fatloss_pk
JWT_SECRET=<your_secret>
REDIS_URL=redis://redis:6379
```

### 8.4 启动命令
```bash
# 构建并启动
docker-compose up -d --build

# 首次初始化数据库表
docker-compose exec backend python -m alembic upgrade head

# 导入食物基础数据
docker-compose exec backend python -m app.seed.foods
```

## 10. 设计规范

### 10.1 色彩
- 主色: #007aff（蓝色）
- 成功/减重: #34c759（绿色）
- 警告/热量: #ff9500（橙色）
- 危险: #ff3b30（红色）
- 背景: #f5f5f7
- 卡片: #ffffff
- 文字: #1d1d1f / 次要 #86868b

### 10.2 布局
- 移动端优先，375px 基准宽度，居中显示
- 卡片圆角: 14-18px
- 卡片阴影: 0 2px 10px rgba(0,0,0,0.03)
- 底部固定导航 5 Tab + 悬浮添加按钮

### 10.3 交互
- 记录操作均通过底部弹窗完成（不跳转页面）
- 数据更新后页面局部刷新，无整页刷新
- 操作成功/失败通过页面内 toast 提示
